# 確率統計解析・データ分析用 標準リポジトリ

本リポジトリは、時系列データ解析、因果推論、および機械学習モデルの構築など、確率統計解析を伴う研究プロジェクトを円滑かつ安全に進行するための標準テンプレートである。

## ディレクトリ構成の詳細

研究の再現性を担保するため、以下のツリー構造と役割分担に従ってファイルを配置すること。

```text
.
├── .env.example          # 環境変数（APIキー等）のひな形（基本いじらなくてOK）
├── .gitignore            # Gitの追跡対象外リスト（ログや大容量データ）
├── README.md             # 本ドキュメント
├── requirements.txt      # 実行に必要なPythonパッケージ一覧
├── configs/              # 実験条件(パラメータとか閾値とか)を管理する設定ファイル群
│   ├── default_params.yaml
│   └── model_params.yaml
├── data/                 # データセット格納領域（Git同期対象外）
│   ├── raw/              # 取得した未加工の生データ（スクリプトでの上書き厳禁）
│   └── processed/        # モデル入力用に整形・前処理されたデータ
├── notebooks/            # データの傾向確認やプロトタイピング用Jupyter Notebook
├── src/                  # 分析システムのコアスクリプト群（本番用コード）
│   ├── models/           # 予測モデルなどのクラス定義
│   ├── utils/            # ログ出力やファイル読み込みなどの共通便利関数
│   ├── data_loader.py    # raw/からデータを読み込みprocessedへ変換する処理
│   ├── eval.py           # 提案手法の精度を算出する処理
│   └── main.py           # メイン実行スクリプト
├── scripts/              # 実験一括実行用スクリプト
│   └── run_all.sh        # 複数条件の実験を全自動で連続実行するシェルスクリプト
├── results/              # 分析結果の出力先
│   ├── metrics/          # 精度スコアなどを記録した.jsonファイル群
│   └── figures/          # 生成された評価のグラフなどの画像群
├── tests/                # 自作した関数の動作確認用テストコード
├── notes/                # 論文原稿には含めないアイデアメモや数式導出過程
└── paper/                # 論文および発表資料のTeXファイル群
    ├── shared/           # 論文とスライドで共通利用する画像や参考文献リスト
    ├── thesis/           # 論文の原稿
    └── slide/            # 発表用スライド
```

## 開発環境の構築手順（Mac環境推奨）

ターミナル操作に不慣れな場合は、以下の手順を1行ずつ順番に実行することで安全に環境を構築できる。

### ステップ1: リポジトリの作成と取得
GitHubの画面右上にある緑色のボタン **Use this template** をクリックし、**Create a new repository** を選択する。自身のアカウント上に専用の研究リポジトリが作成されるため、Macのターミナルを開き、以下のコマンドで手元のPCにデータをダウンロード（クローン）する。

```bash
# 自身のユーザー名とリポジトリ名に書き換えて実行する
git clone [https://github.com/あなたのユーザー名/作成したリポジトリ名.git](https://github.com/あなたのユーザー名/作成したリポジトリ名.git)

# ダウンロードしたフォルダの中へ移動する
cd 作成したリポジトリ名
```

### ステップ2: 独立したPython環境の作成
他の授業や研究のシステムと設定が混ざるのを防ぐため、このフォルダ専用の仮想環境を作成し、有効化する。

```bash
python3 -m venv venv
source venv/bin/activate
```
※ターミナルの入力行の左端に `(venv)` と表示されていれば成功である。

### ステップ3: 必須パッケージのインストール
研究に必要なライブラリ群を一括でインストールする。

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### ステップ4: 機密情報ファイルの設定
以下のコマンドを実行し、環境変数のひな形ファイルを複製する。作成された `.env` ファイルの中に、自身で取得したAPIキーなどを必要に応じて書き込む。

```bash
cp .env.example .env
```

## 分析の実行ワークフロー

実際の研究開発は、以下の5つのフェーズを繰り返して進行する。

1. **データ準備フェーズ**
   分析対象となるCSVファイルなどの生データを `data/raw/` フォルダ内に保存する。
2. **試行錯誤フェーズ**
   ターミナルで `jupyter lab` コマンドを実行し、ブラウザ上で `notebooks/` 内のファイルを開く。ここでデータの可視化や、少量のデータを用いたアルゴリズムの動作テストを行う。
3. **コード整理フェーズ**
   Notebook上で正しく動いた数式やデータ処理のプログラムを、他のメンバーも再利用できるように `src/` 配下のPythonファイル（.py）に清書して移す。
4. **実験実行フェーズ**
   実験で試したい数値（検定の有意水準など）を `configs/` 内のYAMLファイルに記述する。その後、ターミナルからスクリプトを実行し、本格的な計算処理を開始する。
5. **結果検証フェーズ**
   計算が完了すると `results/figures/` にグラフ等の画像が自動生成される。この結果を考察し、必要に応じて手順2や手順4に戻り、手法の改善を繰り返す。
